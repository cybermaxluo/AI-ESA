## 2.8.3 政策文档模板

### 模板1：信息安全政策（通用模板）

```markdown
# 信息安全政策
**文档编号**：POL-SEC-001
**版本**：v2.0
**生效日期**：2025年1月1日
**下次复审日期**：2027年1月1日
**文档所有者**：CISO
**批准人**：CEO

---

## 1. 目的

本政策旨在建立企业信息安全管理框架，保护信息资产的机密性、完整性和可用性，支撑业务目标实现。

## 2. 适用范围

- **人员**：全体员工、承包商、第三方合作伙伴
- **系统**：所有IT系统、应用、数据库
- **数据**：所有企业数据（客户数据、员工数据、业务数据）
- **地理**：全球所有办公室与数据中心

## 3. 角色与职责

| 角色 | 职责 |
|------|------|
| **董事会** | 批准信息安全策略，监督重大风险 |
| **CEO** | 批准信息安全政策，提供资源支持 |
| **CISO** | 制定并执行信息安全计划，向CEO汇报 |
| **业务负责人** | 识别业务风险，确保团队遵守政策 |
| **员工** | 阅读并遵守本政策，及时报告安全事件 |

## 4. 政策声明

### 4.1 信息分类与保护

所有信息必须按照数据分类标准（公开/内部/机密/高度机密）进行分类和保护。

**要求**：
- **MUST**：机密及以上数据必须加密存储和传输
- **MUST**：高度机密数据访问需双因素认证（MFA）
- **SHOULD**：内部数据应限制访问范围

### 4.2 访问控制

采用最小权限原则，用户仅获得履行职责所需的最小访问权限。

**要求**：
- **MUST**：所有访问请求需经过审批
- **MUST**：季度执行访问权限复审
- **MUST**：离职当日撤销所有访问权限

### 4.3 密码与认证

**要求**：
- **MUST**：生产系统强制MFA
- **MUST**：密码最小长度12位，包含大小写+数字+特殊字符
- **MUST**：密码90天强制更换
- **MUST**：禁止重复使用最近12次密码

### 4.4 安全事件报告

**要求**：
- **MUST**：所有安全事件需在发现后1小时内报告给安全团队（security@company.com）
- **MUST**：涉及客户数据的事件需在4小时内上报CISO

## 5. 例外处理

对于无法满足本政策要求的情况，需提交例外申请：
- 填写《安全政策例外申请表》
- 说明业务理由、补偿控制措施
- 提交CISO审批
- 例外有效期最长90天，到期需重新申请

## 6. 合规性

### 6.1 监测
- 自动化工具扫描政策合规性（每日）
- 内部审计抽样检查（季度）

### 6.2 违规后果
违反本政策将导致纪律处分，包括：
- 第一次违规：书面警告
- 重复违规：降职或解雇
- 严重违规（如恶意泄露数据）：解雇并追究法律责任

## 7. 相关文档

- 上级文档：信息安全策略（STR-SEC-001）
- 下级文档：
  - 访问控制标准（STD-IAM-001）
  - 数据加密标准（STD-CRYPTO-001）
  - 密码管理指南（GUIDE-IAM-001）
```

## 2.8.4 KPI/KRI指标体系

### KPI（关键绩效指标）

| 类别               | 指标名称         | 计算公式                        | 目标值  | 监测频率 | 负责人     |
| ------------------ | ---------------- | ------------------------------- | ------- | -------- | ---------- |
| **风险管理** | 高风险关闭率     | 本月关闭高风险数 / 期初高风险数 | ≥80%   | 月度     | CRO        |
| **风险管理** | 风险评估覆盖率   | 已评估业务流程数 / 总业务流程数 | 100%    | 年度     | CRO        |
| **合规**     | 合规项目按期率   | 按期完成项目数 / 总项目数       | ≥90%   | 月度     | 合规负责人 |
| **合规**     | 合规培训完成率   | 已完成培训人数 / 应完成人数     | ≥95%   | 季度     | HR+合规    |
| **合规**     | 审计准备时间     | 从审计通知到准备完成的天数      | ≤14天  | 每次审计 | 合规负责人 |
| **政策**     | 政策复审及时率   | 按期复审政策数 / 到期政策数     | 100%    | 季度     | CISO       |
| **政策**     | 政策签收率       | 已签收人数 / 应签收人数         | ≥98%   | 每次发布 | 合规负责人 |
| **运营**     | 平均事件响应时间 | 总响应时间 / 事件数             | ≤4小时 | 月度     | SOC        |
| **运营**     | 高危漏洞修复时效 | 从发现到修复的平均天数          | ≤30天  | 月度     | 安全工程   |

### KRI（关键风险指标）

| 风险类别           | 指标名称           | 阈值（黄色） | 阈值（红色） | 监测频率 | 数据源       |
| ------------------ | ------------------ | ------------ | ------------ | -------- | ------------ |
| **数据泄露** | 高危漏洞未修复天数 | >30天        | >60天        | 周       | 漏洞管理系统 |
| **数据泄露** | 未加密数据库数量   | >0           | >3           | 周       | CSPM         |
| **可用性**   | 关键系统可用性     | <99.5%       | <99%         | 实时     | 监控系统     |
| **合规**     | 未完成合规项数量   | >10个        | >20个        | 月       | GRC平台      |
| **合规**     | 政策过期数量       | >2个         | >5个         | 月       | 政策门户     |
| **第三方**   | 高风险供应商占比   | >15%         | >25%         | 季度     | TPRM平台     |
| **内部威胁** | 异常数据下载次数   | >50次/周     | >100次/周    | 日       | DLP          |
| **钓鱼攻击** | 钓鱼邮件点击率     | >5%          | >10%         | 演练后   | 钓鱼演练平台 |

## 2.8.5 审计准备清单

### ISO 27001审计准备清单

**审计前4周**：

- [ ] 确认审计范围（边界、系统清单、人员清单）
- [ ] 预定审计日期与审计师
- [ ] 准备审计房间（会议室、网络、投影仪）
- [ ] 通知相关人员审计安排

**审计前2周**：

- [ ] 执行内部预审（模拟审计）
- [ ] 整改预审发现的问题
- [ ] 准备证据包（统一命名、分类存放）
- [ ] 准备14个域的控制证据清单

**审计前1周**：

- [ ] 最终证据检查（确保完整性、时效性）
- [ ] 审计师访谈名单确认
- [ ] 准备审计开场PPT（公司介绍、ISMS范围）
- [ ] 打印纸质证据备份

**审计当天**：

- [ ] 开场会议（审计师介绍、审计计划）
- [ ] 文档审查（政策、流程、记录）
- [ ] 人员访谈（10-15人）
- [ ] 现场检查（数据中心、办公区）
- [ ] 闭幕会议（审计发现、不符合项）

**审计后2周**：

- [ ] 整改不符合项
- [ ] 提交整改证据给审计师
- [ ] 审计师验证整改
- [ ] 颁发证书
